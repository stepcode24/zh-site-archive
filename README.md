# zh-site-archive

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件的形式存储，可通过 GitHub Pages 直接访问。仓库内容不针对特定域名或具体网站，仅作为通用页面存档与分发用途。

## 项目简介

该项目旨在集中存放一批独立的 HTML 文件，每个文件对应一个独立的归档页面。这些页面可能包含各类信息、参考文档或临时内容，以便于快速分享和长期保存。所有页面均为纯静态 HTML，不依赖后端服务。

## 目录说明

- `docs/` — 存放所有归档 HTML 页面，每个文件独立存在。
- `assets/` — 存放页面可能引用的资源文件，如 CSS、图片、字体等。
- `index.html` — 仓库主页，提供页面列表或导航入口。

您可根据需要自行调整目录结构，但建议保持 `docs/` 作为页面主目录，以方便 GitHub Pages 部署。

## 页面归档说明

- 每个 HTML 文件应自包含、可独立打开，不依赖外部动态资源。
- 文件名建议采用简洁、无意义冲突的命名方式（如 `page-001.html`）。
- 如需引用本地资源，请将资源放置于 `assets/` 目录下，并使用相对路径引用。
- 所有页面应避免包含敏感信息、违法内容或侵犯他人权益的材料。

## 维护说明

- 本仓库欢迎通过 Pull Request 提交新的归档页面，或对现有页面进行修正。
- 提交前请确保页面内容完整、无外部链接失效（如有必要，可内嵌资源）。
- 仓库维护者会定期检查页面可用性，但不对内容准确性或时效性作额外承诺。
- 如需删除或替换页面，请直接在对应目录下操作并提交更改。
